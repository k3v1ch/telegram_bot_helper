# Telegram Digest Bot

Многопользовательский бот, который собирает дайджесты из ваших Telegram-чатов. Каждый пользователь подключает **несколько собственных аккаунтов** Telegram через бота, выбирает откуда читать и куда отправлять — и получает аккуратные сводки по расписанию.

## Что умеет

- Парсит Telegram-чаты от имени реального аккаунта пользователя (через Telethon)
- Поддерживает **несколько аккаунтов на пользователя** — каждый чат привязан к конкретному аккаунту
- Анализирует сообщения через AI (Groq, Llama 4 Scout)
- Отправляет структурированный дайджест в указанный топик группы или в личные сообщения
- Поддерживает несколько пользователей одновременно — данные изолированы
- **Per-chat ключевые слова** для алертов в реальном времени
- Поиск по архиву дайджестов
- **Полностью inline-меню** (никаких reply-клавиатур), навигация через редактирование одного сообщения

## Установка

```bash
git clone <repo-url>
cd telegram-digest
cp .env.example .env
# заполнить .env: BOT_TOKEN, ADMIN_USER_ID, GROQ_API_KEY, POSTGRES_PASSWORD
docker compose up -d
```

PostgreSQL стартует первым (healthcheck `pg_isready`), бот ждёт её готовности.

### Что нужно в `.env`

| Переменная | Где взять |
|---|---|
| `BOT_TOKEN` | [@BotFather](https://t.me/BotFather) |
| `ADMIN_USER_ID` | свой numeric Telegram ID — узнать через [@userinfobot](https://t.me/userinfobot) |
| `GROQ_API_KEY` | [console.groq.com](https://console.groq.com/keys), бесплатный тариф |
| `POSTGRES_PASSWORD` | пароль для БД |
| `DATABASE_URL` | по умолчанию `postgresql+asyncpg://digest:password@postgres:5432/digest_bot` |
| `TELEGRAM_API_ID`, `TELEGRAM_API_HASH` | общие credentials для всех пользователей |

## Первый запуск

1. Напишите боту `/start` — он зарегистрирует вас и покажет главное меню.
2. Если у вас ещё нет аккаунтов, главное меню содержит только кнопки **📱 Аккаунты** и (для админа) **👑 Админ**.
3. **📱 Аккаунты → ➕ Добавить аккаунт** → введите название (например, "Основной"), номер телефона, код из Telegram, и пароль 2FA если включён.
4. После подключения хотя бы одного аккаунта появляются кнопки **💬 Мои чаты** и **📊 Статистика**.
5. **💬 Мои чаты → ➕ Добавить чат**: название → выбор аккаунта (если их несколько) → источник → назначение → время → период → промпт.

## Главное меню

```
[📱 Аккаунты]                       ← всегда
[💬 Мои чаты] [📊 Статистика]       ← только если есть авторизованный аккаунт
[👑 Админ]                          ← только ADMIN_USER_ID
```

Без авторизованного аккаунта попытка попасть в Чаты/Статистику показывает попап "⚠️ Сначала подключите аккаунт Telegram".

## Экран аккаунтов

```
👤 Ваши аккаунты (2):

🟢 +79001234567 — Основной       [⚙️]
🟢 +79009876543 — Рабочий        [⚙️]
[➕ Добавить аккаунт]
[← Назад]
```

`🟢` — Telethon-клиент подключён, `🔴` — отключён.

Карточка аккаунта позволяет переименовать, переподключить или удалить аккаунт (привязанные чаты остаются, но без сессии — нужно перепривязать через настройки чата или удалить).

## Куда отправлять дайджест

Поле "назначение" принимает два формата:

- **Группа с топиком:** `-1003332852289:220` — chat_id группы, двоеточие, topic_id топика.
- **Личные сообщения:** просто ваш `user_id`, например `635544292`.

Источник всегда `chat_id:topic_id` (бот читает конкретный топик группы).

## Меню чата

```
📋 Bedolaga БС
📱 Аккаунт: Основной (+79001234567)
🕐 Расписание: 05:00 МСК
📏 Период: 24ч
⚡ Алерты: ВКЛ
🔑 Ключевые слова: выведен из бс, упал, критично…

[▶️ Запустить]
[⏱ 1ч] [⏱ 5ч] [⏱ 12ч] [⏱ 24ч]
[📅 За неделю] [🔎 Поиск]
[⚡ Алерты: ВКЛ] [🔔 Ключевые слова]
[⚙️ Настройки] [⏸ Пауза]
[← Назад]
```

**Настройки чата** позволяют изменить промпт, расписание, период, источник/назначение или удалить чат.

## Per-chat ключевые слова для алертов

Каждый чат имеет свой список ключевых слов. По умолчанию — `выведен из бс, добавлен в бс, упал, лег намертво, критично, срочно`. Изменяется через кнопку **🔔 Ключевые слова → ✏️ Изменить список** (вводится через запятую). Пустой ввод сбрасывает в дефолт.

## Real-time алерты

При включении алертов userbot слушает источник чата в режиме реального времени. В назначение прилетают сообщения, содержащие любое из настроенных ключевых слов. Если рядом с ключевым словом есть IP-адрес — приоритет повышается.

Дебаунс: одинаковые алерты от одного отправителя — 10 минут.

## Структура дайджеста

```
📋 Дайджест • Bedolaga БС
📅 12 мая 2026 • 09:15 – 16:42 МСК
💬 Проанализировано: 87
📊 Вчера: 64 → сегодня: 87 (▲ +23)
🔍 Обработано: 87 → 87 → 32
━━━━━━━━━━━━━━━━━━━━━━

📌 Резюме: [одно предложение — главное за период]

## 🔴 Важное
…

## 🟡 Обновления
…

## 🔵 Полезно
…
```

Если закреп изменился — перед дайджестом приходит "📌 Закреп обновлён" + сам закреп через userbot.

Default Stage 3 prompt универсальный (подходит любому чату); опционально можно задать `custom_prompt` для конкретного чата.

## Для администратора

`ADMIN_USER_ID` видит в главном меню **👑 Админ**:

- Список пользователей с пагинацией
- Блокировка/разблокировка (останавливает все сессии пользователя + cron-задачи)
- Глобальная статистика
- Перезапуск всех userbot-сессий
- Переподключение сессий конкретного пользователя

## Логи

`./logs/digest.log` — ротация 5 МБ × 3 файла. То же в stdout: `docker compose logs -f digest`.

## Архитектура (кратко)

- `bot/main.py` — entry point: poll, scheduler, alerters, миграция legacy env, error handler.
- `bot/userbot/manager.py` — `UserbotManager`: N Telethon-клиентов по `session_id`, `keep_alive`.
- `bot/scheduler.py` — `DigestScheduler`: daily + Monday weekly cron на каждый активный чат + on-demand.
- `bot/analyzer.py` — Groq API: 3 стадии (фильтр → сжатие → дайджест), generic prompts.
- `bot/db/` — SQLAlchemy 2.0 async + asyncpg, PostgreSQL 16.
- `bot/handlers/` — python-telegram-bot v20+, **только inline** клавиатуры.
- `bot/keyboards.py` — все клавиатуры через `InlineKeyboardMarkup`.

Подробнее — см. [`CLAUDE.md`](./CLAUDE.md).
