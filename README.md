# Telegram Digest Bot

Юзербот + бот для мониторинга Telegram-группы. Юзербот (Telethon) только читает сообщения из исходного чата. Бот (python-telegram-bot) отправляет дайджесты, алерты и предоставляет inline-меню для управления.

## Что делает

- Читает сообщения за последние 24 часа из указанного топика группы (юзербот)
- Анализирует через Groq API и формирует сводку с кратким резюме дня
- Публикует дайджест в целевой топик (бот)
- Работает по расписанию (по умолчанию — 09:00 MSK)
- Еженедельный дайджест по понедельникам (настраивается)
- Мониторит закреплённое сообщение и уведомляет об изменениях
- Сравнивает количество сообщений с предыдущим днём
- Отправляет real-time алерты при критических сообщениях (бот)
- Поиск по архиву дайджестов
- Клавиатура с кнопками для управления (дайджест по периодам, недельный, поиск, статус, алерты)

## Архитектура

- **Telethon (юзербот)** — только читает SOURCE-чат, никогда не отправляет сообщений
- **python-telegram-bot (бот)** — отправляет дайджесты, алерты, обрабатывает команды
- **Groq API** — анализ сообщений
- Все три компонента работают в одном процессе через `asyncio.gather`

## Требования

1. **Telegram API credentials** — получить на [my.telegram.org](https://my.telegram.org):
   - `API_ID` и `API_HASH`

2. **Groq API key** — получить на [console.groq.com](https://console.groq.com/keys) (бесплатный тариф)

3. **Telegram Bot Token** — создать бота у [@BotFather](https://t.me/BotFather)

4. **ADMIN_USER_ID** — ваш числовой Telegram ID (узнать у [@userinfobot](https://t.me/userinfobot))

5. **Docker** и **docker-compose**

## Формат SOURCE и DEST

Оба параметра задаются в формате `chat_id:topic_id`:

```
SOURCE=-1003332852289:155
DEST=-1003332852289:220
```

### Как узнать chat_id

- Перешлите сообщение из группы боту [@userinfobot](https://t.me/userinfobot) — он покажет числовой ID

### Как узнать topic_id

1. Откройте группу в [Telegram Web](https://web.telegram.org)
2. Перейдите в нужный топик
3. ID топика — число после `/topics/` в URL

## Установка и запуск

### 1. Клонировать и настроить

```bash
git clone <repo-url>
cd telegram-digest
cp .env.example .env
# Заполнить .env своими данными
```

### 2. Первый запуск (авторизация Telethon)

При первом запуске Telethon запросит номер телефона и код подтверждения:

```bash
docker compose run digest
```

После авторизации сессия сохранится в `./sessions/`.

### 3. Запуск в фоне

```bash
docker compose up -d
```

## Управление ботом

Команды:
- `/start`, `/menu` — открыть меню
- `/help` — справка
- `/cancel` — выйти из режима поиска
- `/digest <period>` — дайджест за произвольный период (`6h`, `1d`, `48`, `3д`, диапазон 1–168 ч)
- `/cleanup [days]` — удалить дайджесты старше N дней (по умолчанию 90)

Отправьте `/start` или `/menu` боту в личные сообщения. Появится клавиатура:

| Кнопка | Действие |
|--------|----------|
| **⏱ 1 час / 5 часов / 12 часов / 24 часа** | Дайджест за выбранный период |
| **📅 За неделю** | Еженедельный дайджест за 7 дней |
| **🔎 Поиск** | Поиск по архиву дайджестов |
| **📊 Статус** | Время последнего/следующего запуска, состояние юзербота |
| **⚡ Алерты** | Переключение real-time алертов |

Доступ только для `ADMIN_USER_ID`. Все остальные получают "⛔ Доступ запрещён".

## Ручной запуск (CLI)

```bash
docker compose run digest --now
```

## Real-time алерты

Юзербот мониторит SOURCE-чат в реальном времени. Бот отправляет алерт в DEST при:

- Упоминании IP-адреса вместе с ключевыми словами (выведен, добавлен, заблокирован и т.д.)
- Высокоприоритетных фразах: "выведен из бс", "добавлен в бс", "упал", "критично", "срочно"

Дебаунс: одинаковые алерты от одного отправителя игнорируются 10 минут. Алерты можно включить/выключить через inline-меню.

## Структура дайджеста

- 📌 **Резюме** — одно предложение, самое главное за день
- 🔴 **Критично** — IP выведены/добавлены в БС, критические изменения
- 🟡 **Обновления** — проверенные факты о провайдерах, IP-диапазонах, ценах
- 🔵 **Полезно** — технические выводы, рабочие конфиги, рекомендации

Пустые секции не отображаются. Если закреплённое сообщение изменилось — отображается в шапке.

## Еженедельный дайджест

Автоматически генерируется по расписанию (по умолчанию — понедельник 05:00 MSK). День настраивается через `WEEKLY_DIGEST_DAY` (monday, tuesday, ...). Также можно запустить вручную кнопкой **📅 За неделю**.

## Поиск по дайджестам

Все дайджесты сохраняются в `./data/digests/`. Кнопка **🔎 Поиск** ищет по ключевому слову, показывает до 5 результатов с совпадающими строками.

## Хранение данных

- `./data/stats.json` — количество сообщений по дням
- `./data/state.json` — состояние бота (алерты, последний запуск, следующий запуск)
- `./data/pinned.json` — последнее закреплённое сообщение
- `./data/digests/` — архив всех дайджестов в формате `YYYY-MM-DD_HHMM_{period}.json`
- `./sessions/` — сессия Telethon

Все JSON-записи делаются атомарно через temp-файл + `os.replace`, поэтому крэш процесса во время записи не повредит данные.

## Надёжность

- Параллельные дайджесты невозможны: внутри `run_digest` есть `asyncio.Lock`, второй вызов ждёт окончания первого
- У Groq-вызовов настроен таймаут 60 секунд и повторная попытка через 30 секунд при ошибке 429
- У scheduler-jobs включены `misfire_grace_time` и `coalesce=True` — после простоя ничего не задублируется
- Ошибки, отправляемые в чат, обрезаются до 300 символов и без переноса строк
- Старые дайджесты автоматически удаляются (старше 90 дней) при каждом сохранении нового

## Healthcheck

Бот поднимает HTTP-сервер на порту `HEALTH_PORT` (по умолчанию 8080):

- `GET /health` — JSON со статусом, состоянием юзербота, временем последнего/следующего запуска. Возвращает 200 если юзербот подключён, 503 если нет.
- `GET /ping` — простой `pong` для liveness-проб.

В Dockerfile настроен `HEALTHCHECK` через curl. Docker-контейнер будет помечаться `unhealthy` если юзербот отвалился.

## Тесты

```bash
pip install -r requirements-dev.txt

```

96 unit-тестов покрывают парсинг, фильтрацию, сохранение/поиск дайджестов, атомарную запись, состояние, алерты, healthcheck. Внешние сервисы (Telegram, Groq) не вызываются.
