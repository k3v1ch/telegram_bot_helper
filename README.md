# Telegram Digest Bot

Юзербот, который ежедневно собирает сообщения из указанного топика Telegram-группы, отправляет их в Groq для анализа и публикует структурированный дайджест в ваш личный чат/топик.

## Что делает

- Читает сообщения за последние 24 часа из указанного топика группы
- Отправляет лог в Groq API (Llama 3.3 70B) для формирования сводки
- Публикует дайджест с группировкой по важности в целевой топик
- Работает по расписанию (по умолчанию — 09:00 MSK)
- Автоматически сканирует упоминания IP-адресов, провайдеров и ключевых слов
- Сравнивает количество сообщений с предыдущим днём
- (Опционально) Отправляет real-time алерты при критических сообщениях
- (Опционально) Поддерживает команду `/digest` через бот-токен

## Требования

1. **Telegram API credentials** — получить на [my.telegram.org](https://my.telegram.org):
   - `API_ID` и `API_HASH`

2. **Groq API key** — получить на [console.groq.com](https://console.groq.com/keys) (бесплатный тариф)

3. **Docker** и **docker-compose**

4. (Опционально) **Telegram Bot Token** — для команды `/digest`, получить у [@BotFather](https://t.me/BotFather)

## Формат SOURCE и DEST

Оба параметра задаются в формате `chat_id:topic_id`, например:

```
SOURCE=-1003332852289:155
DEST=-1003332852289:220
```

### Как узнать chat_id

- Перешлите сообщение из группы боту [@userinfobot](https://t.me/userinfobot) — он покажет числовой ID (например `-1001234567890`)

### Как узнать topic_id

1. Откройте группу в [Telegram Web](https://web.telegram.org)
2. Перейдите в нужный топик
3. ID топика — число после `/topics/` в URL, либо `message_id` первого сообщения в топике

## Установка и запуск

### 1. Клонировать и настроить

```bash
git clone <repo-url>
cd telegram-digest
cp .env.example .env
# Заполнить .env своими данными
```

### 2. Первый запуск (авторизация)

При первом запуске Telethon запросит номер телефона и код подтверждения. Запускайте интерактивно:

```bash
docker compose run digest
```

После успешной авторизации сессия сохранится в `./sessions/`.

### 3. Запуск в фоне

```bash
docker compose up -d
```

## Ручной запуск дайджеста

```bash
docker compose run digest --now
```

## Команда /digest

Если указан `BOT_TOKEN` в `.env`, бот запускается параллельно с юзерботом. Отправьте `/digest` вашему боту в Telegram — он немедленно сгенерирует и отправит дайджест в целевой топик.

Для ограничения доступа укажите `ADMIN_ID` — числовой Telegram ID пользователя, которому разрешено вызывать `/digest`. Все остальные будут игнорироваться. Узнать свой ID можно у [@userinfobot](https://t.me/userinfobot). Если `ADMIN_ID` не задан — команда доступна всем.

## Real-time алерты

Установите `ALERTS_ENABLED=true` в `.env`. Юзербот будет в реальном времени мониторить SOURCE-чат и отправлять алерт в DEST при:

- Упоминании IP-адреса вместе с ключевыми словами (выведен, добавлен, заблокирован и т.д.)
- Высокоприоритетных фразах: "выведен из бс", "добавлен в бс", "упал", "критично", "срочно"

Дебаунс: одинаковые алерты от одного отправителя игнорируются 10 минут.

## Структура дайджеста

Бот формирует дайджест с разделами:

- 🔴 **Важное** — IP выведены/добавлены в БС, критические изменения
- 🟡 **Обновления** — проверенные факты о провайдерах, IP-диапазонах, ценах
- 🔵 **Полезное** — технические выводы, рабочие конфиги, рекомендации
- 🔍 **Упоминания** — автоматический скан IP-адресов, провайдеров и ключевых слов

Пустые секции не отображаются. Если чат был тихим — отправляется короткое уведомление.

## Статистика

Количество сообщений за каждый день сохраняется в `./data/stats.json`. В заголовке дайджеста показывается сравнение с предыдущим днём.

## Логи

Логи пишутся в `./logs/digest.log` с ротацией (макс. 5 МБ, 3 файла).

Просмотр в реальном времени:

```bash
docker compose logs -f digest
```
