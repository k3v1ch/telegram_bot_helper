# Telegram Digest Bot

Юзербот + бот для мониторинга Telegram-группы. Юзербот (Telethon) только читает сообщения из исходного чата. Бот (python-telegram-bot) отправляет дайджесты, алерты и предоставляет inline-меню для управления.

## Что делает

- Читает сообщения за последние 24 часа из указанного топика группы (юзербот)
- Анализирует через Groq API (Llama 3.3 70B) и формирует сводку
- Публикует дайджест в целевой топик (бот)
- Работает по расписанию (по умолчанию — 09:00 MSK)
- Автоматически сканирует упоминания IP-адресов, провайдеров и ключевых слов
- Сравнивает количество сообщений с предыдущим днём
- Отправляет real-time алерты при критических сообщениях (бот)
- Inline-меню с кнопками для управления (запуск дайджеста, статус, алерты)

## Архитектура

- **Telethon (юзербот)** — только читает SOURCE-чат, никогда не отправляет сообщений
- **python-telegram-bot (бот)** — отправляет дайджесты, алерты, обрабатывает команды
- **Groq API** — анализ сообщений
- Все три компонента работают в одном процессе через `asyncio.gather`

## Требования

1. **Telegram API credentials** — получить на [my.telegram.org](https://my.telegram.org):
   - `API_ID` и `API_HASH`

2. **Groq API key** — получить на [console.groq.com](https://console.groq.com/keys) (бесплатный тариф)

3. **Telegram Bot Token** — создать бота у [@BotFather](https://t.me/BotFather)

4. **ADMIN_USER_ID** — ваш числовой Telegram ID (узнать у [@userinfobot](https://t.me/userinfobot))

5. **Docker** и **docker-compose**

## Формат SOURCE и DEST

Оба параметра задаются в формате `chat_id:topic_id`:

```
SOURCE=-1003332852289:155
DEST=-1003332852289:220
```

### Как узнать chat_id

- Перешлите сообщение из группы боту [@userinfobot](https://t.me/userinfobot) — он покажет числовой ID

### Как узнать topic_id

1. Откройте группу в [Telegram Web](https://web.telegram.org)
2. Перейдите в нужный топик
3. ID топика — число после `/topics/` в URL

## Установка и запуск

### 1. Клонировать и настроить

```bash
git clone <repo-url>
cd telegram-digest
cp .env.example .env
# Заполнить .env своими данными
```

### 2. Первый запуск (авторизация Telethon)

При первом запуске Telethon запросит номер телефона и код подтверждения:

```bash
docker compose run digest
```

После авторизации сессия сохранится в `./sessions/`.

### 3. Запуск в фоне

```bash
docker compose up -d
```

## Управление ботом

Отправьте `/start` или `/menu` боту в личные сообщения. Появится inline-меню:

- **🔄 Запустить дайджест** — немедленная генерация и отправка
- **📊 Статус** — время последнего/следующего запуска, количество сообщений, состояние юзербота
- **⚡ Алерты: ВКЛ/ВЫКЛ** — переключение real-time алертов

Доступ только для `ADMIN_USER_ID`. Все остальные получают "⛔ Доступ запрещён".

## Ручной запуск (CLI)

```bash
docker compose run digest --now
```

## Real-time алерты

Юзербот мониторит SOURCE-чат в реальном времени. Бот отправляет алерт в DEST при:

- Упоминании IP-адреса вместе с ключевыми словами (выведен, добавлен, заблокирован и т.д.)
- Высокоприоритетных фразах: "выведен из бс", "добавлен в бс", "упал", "критично", "срочно"

Дебаунс: одинаковые алерты от одного отправителя игнорируются 10 минут. Алерты можно включить/выключить через inline-меню.

## Структура дайджеста

- 🔴 **Важное** — IP выведены/добавлены в БС, критические изменения
- 🟡 **Обновления** — проверенные факты о провайдерах, IP-диапазонах, ценах
- 🔵 **Полезное** — технические выводы, рабочие конфиги, рекомендации
- 🔍 **Упоминания** — автоматический скан IP-адресов и провайдеров

Пустые секции не отображаются.

## Хранение данных

- `./data/stats.json` — количество сообщений по дням
- `./data/state.json` — состояние бота (алерты, последний запуск, следующий запуск)
- `./sessions/` — сессия Telethon

## Логи

Логи пишутся в `./logs/digest.log` с ротацией (макс. 5 МБ, 3 файла).

```bash
docker compose logs -f digest
```
