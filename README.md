# Telegram Digest Bot

Многопользовательский бот, который собирает дайджесты из ваших Telegram-чатов. Каждый пользователь подключает свой собственный аккаунт Telegram через бот, выбирает откуда читать и куда отправлять — и получает аккуратные сводки по расписанию.

## Что умеет

- Парсит Telegram-чаты от имени реального аккаунта пользователя (через Telethon)
- Анализирует сообщения через AI (Groq, Llama 3.3 70B)
- Отправляет структурированный дайджест в указанный топик группы или в личные сообщения
- Поддерживает несколько пользователей и несколько чатов одновременно — данные изолированы
- Алерты в реальном времени по ключевым словам и упоминаниям IP-адресов
- Поиск по архиву дайджестов
- Inline-меню для управления, без CLI

## Установка

```bash
git clone <repo-url>
cd telegram-digest
cp .env.example .env
# заполнить .env: BOT_TOKEN, ADMIN_USER_ID, GROQ_API_KEY, POSTGRES_PASSWORD
docker compose up -d
```

PostgreSQL стартует первым (healthcheck `pg_isready`), бот ждёт её готовности.

### Что нужно положить в `.env`

| Переменная | Где взять |
|---|---|
| `BOT_TOKEN` | [@BotFather](https://t.me/BotFather) |
| `ADMIN_USER_ID` | свой numeric Telegram ID — узнать через [@userinfobot](https://t.me/userinfobot) |
| `GROQ_API_KEY` | [console.groq.com](https://console.groq.com/keys), бесплатный тариф |
| `POSTGRES_PASSWORD` | любой надёжный пароль для БД |
| `DATABASE_URL` | по умолчанию `postgresql+asyncpg://digest:password@postgres:5432/digest_bot` |
| `TELEGRAM_API_ID`, `TELEGRAM_API_HASH` | общие credentials для всех пользователей (значения в `.env.example` подходят) |

## Первый запуск

1. Напишите боту `/start` — он зарегистрирует вас.
2. Нажмите **📱 Аккаунт → ➕ Подключить аккаунт**, введите номер телефона, код из Telegram (и пароль 2FA, если включён).
3. Нажмите **💬 Мои чаты → ➕ Добавить чат**, заполните: название, источник, назначение, время отправки, период анализа, при желании — кастомный промпт.
4. Готово. Дайджест будет приходить по расписанию, или можно запустить вручную из меню чата.

## Куда отправлять дайджест

Поле "назначение" принимает два формата:

- **Группа с топиком:** `-1003332852289:220` — `chat_id` группы, двоеточие, `topic_id` топика.
- **Личные сообщения:** просто ваш `user_id`, например `635544292`. Свой ID можно узнать через [@userinfobot](https://t.me/userinfobot).

Источник всегда указывается в формате `chat_id:topic_id` (бот читает конкретный топик группы).

## Как пригласить друга

Отправьте ему ссылку на бота. Он сам сделает `/start`, подключит свой аккаунт через меню и настроит свои чаты. Его аккаунты, сессии, чаты и архив дайджестов полностью изолированы от ваших.

## Для администратора

Пользователь, чей `user_id` указан в `ADMIN_USER_ID`, видит в главном меню кнопку **👑 Админ панель**:

- Просмотр всех пользователей с пагинацией
- Блокировка / разблокировка пользователя — при блокировке его userbot-сессия и cron-задачи останавливаются
- Глобальная статистика
- Перезапуск всех userbot-сессий (`Перезапустить всё`)
- Переподключение сессии конкретного пользователя

## Меню чата

После выбора чата из списка появляются кнопки:

| Кнопка | Действие |
|---|---|
| **▶️ Запустить** | дайджест за дефолтный период (24ч) |
| **⏱ 1ч / 5ч / 12ч / 24ч** | дайджест за выбранный интервал |
| **📅 За неделю** | еженедельный дайджест (168ч) с более компактным форматом |
| **🔎 Поиск** | поиск по архиву дайджестов этого чата |
| **⚡ Алерты: ВКЛ/ВЫКЛ** | переключить real-time алерты |
| **⚙️ Настройки** | изменить промпт / время / период / источник-назначение / удалить |
| **⏸ Пауза / Запустить** | приостановить или возобновить расписание |

## Real-time алерты

При включении алертов userbot слушает источник чата в режиме реального времени. В назначение прилетают сообщения с:

- упоминанием IP-адреса вместе с ключевыми словами ("выведен", "добавлен", "заблокирован" и т.п.);
- ключевыми фразами высокого приоритета ("выведен из бс", "упал", "критично", "срочно", …).

Дебаунс: одинаковые алерты от одного отправителя игнорируются 10 минут.

## Структура дайджеста

Шапка содержит:

```
📋 Дайджест • <название чата>
📅 12 мая 2026 • 09:15 – 16:42 МСК
💬 Проанализировано сообщений: 87
📊 Вчера: 64 → сегодня: 87 (▲ +23)
🔍 Обработано: 87 → 87 → 32
━━━━━━━━━━━━━━━━━━━━━━
```

Дальше — секции дайджеста, сформированные AI (Критично / Обновления / Полезно для дефолтного промпта, либо что попросите в кастомном).

Если закреплённое сообщение в источнике поменялось — перед дайджестом приходит "📌 Закреп обновлён" + сам закреп через userbot.

## Логи

`./logs/digest.log` — ротация 5 МБ × 3 файла. То же самое в stdout: `docker compose logs -f digest`.

## Архитектура (кратко)

- `bot/main.py` — entry point, поднимает БД, миграцию старых env-переменных, UserbotManager, DigestScheduler, alerter'ы для активных чатов и polling Telegram-бота.
- `bot/userbot/manager.py` — `UserbotManager`: жизненный цикл Telethon-клиентов по `user_id`, `keep_alive` каждые 5 минут.
- `bot/scheduler.py` — `DigestScheduler`: cron-задачи (ежедневная + еженедельная) на каждый чат, единый `run_digest` для авто- и ручных запусков.
- `bot/analyzer.py` — Groq API: фильтр → конденсация → финальная сводка.
- `bot/db/` — SQLAlchemy 2.0 async + asyncpg, PostgreSQL 16.
- `bot/handlers/` — `python-telegram-bot` v20+ ConversationHandlers и callbacks.

Подробнее — см. [`CLAUDE.md`](./CLAUDE.md).
